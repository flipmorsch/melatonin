## Anti-Patterns Verdict

## Cognitive Load Assessment

2 failures: no search/filter in sidebar (#6), working memory needed for active context (#7).

## Strengths

1. **One-signature discipline.** Dusk horizon is the only decorative flourish. Method hues are strictly information. Night-light pulse conveys state, not decoration.
2. **Consistent component vocabulary.** MethodBadge, SidebarRow, EmptyState, ConfirmDelete, SectionLabel, CodeEditor reused everywhere.
3. **Auto-save, no modals.** Compose-send-inspect loop is uninterrupted. No Save button. No confirmation dialogs except destructive actions.
4. **Empty states that direct action.** Every empty surface says what to do, not what's missing.

## Priority Issues

### [P1] No loading/request-in-flight indicator on Send
No visual feedback that a request is in flight. Users click Send multiple times or assume the tool is broken on slow connections.
**Fix:** Spinner in Send button, elapsed time display.
**Suggested:** /impeccable polish

### [P1] No autosave status indicator
No indication of whether changes are saved or pending. Closing the app 500ms after editing loses the last change.
**Fix:** Saved/Saving... status dot or text.
**Suggested:** /impeccable polish

### [P2] No keyboard shortcuts
The target user is a terminal-native engineer. No Cmd+Enter, no Cmd+K palette, no keyboard navigation.
**Fix:** Cmd+Enter → Send, Cmd+S → save, Cmd+K → palette.
**Suggested:** /impeccable harden

### [P2] No search/filter in the sidebar
As collections grow, scanning a flat list is expensive. No filter, no search, no jump-to.
**Fix:** Filter input or Cmd+K command palette.
**Suggested:** /impeccable harden

### [P2] Row-reveal delete buttons are mouse-only
Visibility: hidden → visible on :hover. Keyboard users can never reveal or activate.
**Fix:** :focus-within or context menu.
**Suggested:** /impeccable audit
